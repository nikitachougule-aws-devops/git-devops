## 🔀 Branching

The **main (or master)** branch is the final, stable version of the code. It is ready to use or deploy.

A **branch** is like creating a separate path to work on something new without affecting the main code.

When you create a branch, Git does not copy files. It simply creates a new pointer to work independently.

---

##  Why Developers Create Branches

- To add new features  
- To fix bugs  
- To try new ideas safely  

---

## 🔀 Merging

After finishing work on a branch:

- Code is tested  
- Then it is merged into the main branch  

---

##  Types of Merging

- **Fast-forward merge** → Simple, straight history  
- **Merge commit** → Keeps branch history  
- **Rebase + merge** → Clean and linear history
