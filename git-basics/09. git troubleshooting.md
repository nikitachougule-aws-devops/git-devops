# 🛠️ git troubleshooting 

## 🔥 1. Merge Conflicts

### 🔹 What is a Merge Conflict?

A merge conflict occurs when Git cannot automatically combine changes from two branches.

### Example Scenario:

- You modify a file  
- Another developer modifies the same part  
- Git doesn’t know which version to keep  

---

### 🔹 Conflict Markers

```text
<<<<<<< HEAD
your changes
=======
incoming changes
>>>>>>> branch-name

```
## 🔹 How to Resolve Merge Conflicts

- Open the conflicted file
- Choose:
  - Your changes
  - Incoming changes
  - Or combine both
- Remove conflict markers
- Mark as resolved and commit

---

## 🔥 2. Detached HEAD

### 🔹 What is Detached HEAD?

Normally, `HEAD` points to a branch.  
In a detached HEAD state, it points directly to a specific commit.

---

### 🔹 How It Happens

- Checking out a commit instead of a branch  
- Viewing past history  

---

### 🔹 Problem

- Commits are not attached to any branch  
- Can be lost easily  

---

### 🔹 How to Fix

```bash
git checkout -b new-branch

```
## 🔥 3. Recover Lost Commits (reflog)

### 🔹 What is Reflog?

Reflog records every change to `HEAD`, even if commits are no longer visible.

---

### 🔹 Why It’s Powerful

Even if you:

- Delete a branch  
- Reset commits  
- Lose history  

👉 You can still recover it  

---

### 🔹 How It Works

```bash
git reflog

```
### Recovery Process
git checkout -b recovered-branch <commit-hash>

## 🔥 4. Fixing Wrong Commits

Mistakes happen frequently. Git provides multiple ways to fix them.

---

### 🔹 Case 1: Wrong Commit Message

```bash
git commit --amend

```
### 🔹Case 2: Forgot to Add File

```bash
git add file.txt
git commit --amend

```
### 🔹 Case 3: Wrong Last Commit (Local)

```bash
git reset --soft HEAD~1

```
### 🔹 Case 4: Wrong Commit Already Pushed

```bash
git revert <commit>

```
### 🔹 Case 5: Multiple Wrong Commits

```bash
git rebase -i HEAD~n

```
### 🔹 Decision Guide

- Situation	Solution

- File mistake (not committed)	restore

- Last commit mistake (local)	amend / reset

- Multiple commits cleanup	rebase -i

- Pushed mistake	revert

- Lost commits	reflog

### 🔹 Golden Rules

- Never panic — Git rarely deletes data permanently

- Avoid --hard unless you understand the impact

- Use revert for shared branches

- Use reflog when things go wrong

### 📌 Final Insight

Troubleshooting Git is about understanding:

- Where your changes are (working / staging / commit)

- Whether history is shared or local
