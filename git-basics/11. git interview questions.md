## Git Interview Questions and Answers

---

### 1. Basic Git Questions

### What is Git?

Git is a **distributed version control system (DVCS)** used to track changes in source code, collaborate with teams, and manage different versions of a project efficiently.

---

### Difference between Git and GitHub

| Git | GitHub |
|----|----|
| Version control system | Cloud hosting platform |
| Works locally | Works online |
| Manages code history | Provides collaboration tools (PRs, issues, CI/CD) |

---

### What is a repository in Git?

A **repository (repo)** is a project directory where Git tracks files and their version history.

It contains a hidden **`.git` folder** that stores metadata.

---

### What is the staging area (index)?

The **staging area** is an intermediate area where changes are prepared before committing.

#### Workflow

Working Directory → Staging Area → Repository

#### Example

```bash
git add file.txt
git commit -m "Added file"
```
---

### What is a commit in Git?

A commit is a snapshot of the staged changes with a message describing what changed.

#### Example:
```bash
git commit -m "Fix login bug"
```
---

### What does git init do?

Initializes a new Git repository by creating the .git directory.
```bash
git init
```
---

### What does git clone do?

Creates a local copy of a remote repository.
```bash
git clone https://github.com/user/repo.git
```
---
### Purpose of git status

Shows:

- modified files
- staged files
- untracked files
- current branch
```bash
git status
```
---

### Difference between git add and git commit

| Command | Purpose |
|----|----|
| git add | Moves changes to staging area |
| git commit | Saves staged changes to repo history|

---

### What is .gitignore?

A file used to tell Git which files/folders to ignore.

#### Example:
```bash
node_modules/
.env
*.log
```
---

### 2. Branching

### What is a branch in Git?

A branch is a separate line of development.

Default branch:
```bash
main
```
---

### Why do we use branches?


- Develop features independently
- Avoid breaking main code
- Parallel development

---
### Difference between main and develop

| Branch | Purpose |
|----|----|
| main | Production-ready code |
| develop | Integration branch for features |

---

### How to create a branch
```bash
git branch feature-login
```

Or create and switch:
```bash
git checkout -b feature-login
```
---
### Switch between branches
```bash
git checkout branch-name
```
or
```bash
git switch branch-name
```
---

### What is a feature branch?

A branch created to develop a new feature.

#### Example:
```bash
feature/payment-system
```
---

### What is a hotfix branch?

A branch used to quickly fix production bugs.

#### Example:
```bash
hotfix/login-crash
```
---

### What is GitFlow?

GitFlow is a branching strategy that uses structured branches:

- main
- develop
- feature
- release
- hotfix

Used for structured project management.

---

### 3. Merging

### What is git merge?

Combines changes from one branch into another.

#### Example:
```bash
git merge feature-login
```
---
### What is a fast-forward merge?

Occurs when the target branch has no new commits.

Git simply moves the branch pointer forward.

---
### What is a merge commit?

A commit created when two branches are merged with different histories.

It has two parent commits.

---

### What is a merge conflict?

Occurs when two branches modify the same line of code.

---

### How to resolve merge conflicts
Steps:
- Open conflicted file
- Edit and choose correct code
- Remove conflict markers
- Add file
- Commit
```bash
git add file
git commit
```
---

#### 4. Rebase

### What is git rebase?

Reapplies commits from one branch on top of another branch.
```bash
git rebase main
```
---

### Difference between merge and rebase

| Merge | Rebase |
|----|----|
| Preserves history | Rewrites history |
| Creates merge commit | Linear history |
| Safe for shared branches | Risky for shared branches |

---

### What is interactive rebase?

Used to edit, squash, or reorder commits.
```bash
git rebase -i HEAD~3
```
Options:

- pick
- reword
- squash
- drop
---

### When should you use rebase?

Use rebase when:

- Cleaning commit history
- Updating feature branch before merge
---

### Risks of rebasing shared branches

Rebase rewrites commit history which can break collaboration if others already pulled those commits.

---

#### 5. Undoing Changes

### What does git reset do?

Moves the current branch pointer to a previous commit.

---

### Difference between reset types

| Command | Effect |
|----|----|
| soft | Keeps staged changes |
| mixed | Keeps working directory changes |
| hard | Deletes all changes |

#### Example:
```bash
git reset --hard HEAD~1
```
---

### What is git revert?

Creates a new commit that reverses a previous commit.
```bash
git revert commit-id
```
---

### Reset vs Revert

| Reset | Revert |
|----|----|
| Deletes history | Preserves history |
| Dangerous on shared repos | Safe for shared repos |

---

### What is git checkout used for?

Used to:

- Switch branches
- Restore files
```bash
git checkout main
```
---

#### 6. Logs and History

### What is git log?

Displays commit history.
```bash
git log
```
---

### What is git reflog?

Shows all reference changes, even deleted commits.
```bash
git reflog
```
---

### How to recover a deleted commit
1. Check reflog
```bash
git reflog
```
3. Reset to commit
```bash
git reset --hard commit-id
```
---

#### 7. Remote Repositories

### What is a remote repository?

A version of the repository stored on a server.

#### Examples:

- GitHub
- GitLab
- Bitbucket

---

### What does origin mean in Git?

origin is the default name for the remote repository.

---

### What does git push do?

Uploads local commits to the remote repository.
```bash
git push origin main
```
---

### What does git pull do?

Fetch + merge remote changes.
```bash
git pull origin main
```
---

### Fetch vs Pull

| Fetch | Pull |
|----|----|
| Downloads changes | Downloads + merges |
| Safe | May cause conflicts |

---

#### 8. Tags and Releases

### What are Git tags?

Tags mark specific commits, usually for releases.

#### Example:

- v1.0
- v2.1
---

### Annotated vs Lightweight tags

| Annotated | Lightweight |
|----|----|
| Stored as full object | Simple pointer |
| Has metadata | No metadata |

---
### How to create a tag
```bash
git tag v1.0
```

Annotated:
```bash
git tag -a v1.0 -m "Release version 1.0"
```
---

#### 9. Collaboration

### What is a Pull Request (PR)?

A request to merge code changes into another branch, usually reviewed before merging.

---
### What is a code review?

Process where teammates review code before merging to ensure:

- code quality
- fewer bugs
- maintainability
---

### What is a fork in GitHub?

A copy of someone else's repository in your GitHub account.

Used for contributing to open-source projects.

---

### Fork vs Clone

| Fork | Clone |
|----|----|
| Server-side copy | Local copy |
| On GitHub account | On your computer |

---

#### 10. Advanced Git

### What is Git stash?

Temporarily saves uncommitted changes.
```bash
git stash
```
Restore:
```bash
git stash pop
```
---

### What is Git cherry-pick?

Applies a specific commit from another branch.
```bash
git cherry-pick commit-id
```


#### Bonus Practical Interview Questions

### How to resolve a merge conflict?
```bash
git merge branch-name
```
Fix conflict manually → add → commit
```bash
git add file
git commit
```
---

### How to undo the last commit

Keep changes:
```bash
git reset --soft HEAD~1
```
Remove completely:
```bash
git reset --hard HEAD~1
```
Recover lost commit using reflog
```bash
git reflog
git reset --hard commit-id
```
---

### How to squash commits

Interactive rebase:
```bash
git rebase -i HEAD~3
```
Change:
```bash
pick → squash
```
---

### Rename a branch

Rename current branch:
```bash
git branch -m new-name
```
Rename another branch:
```bash
git branch -m old-name new-name
```
