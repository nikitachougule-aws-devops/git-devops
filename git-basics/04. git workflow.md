## 🔄 Git Workflow

A **basic Git workflow** is the standard sequence developers follow to track changes, collaborate, and manage code safely.

</p>
 
### 1. WORKING DIRECTORY
- Contains actual project files
- You create, edit, delete files here
- Changes are NOT tracked yet

#### Example:
Edit index.html → still in working directory

---
### 2. STAGING AREA (INDEX)
- Temporary area to prepare changes for commit
- Select specific files to include

#### Command:
git add <file>

#### Meaning:
Tell Git to include this file in the next commit

---
### 3. LOCAL REPOSITORY
- Stores committed changes on your system
- Each commit = snapshot of project

#### Command:
git commit -m "message"

#### Meaning:
Save staged changes permanently

---

### 4. REMOTE REPOSITORY
- Hosted online (e.g., GitHub)
- Used for backup and collaboration

#### Commands:
git push   → upload code
git pull   → download latest changes

---

### FULL WORKFLOW
Edit → Stage → Commit → Push

---

### ONE-LINE SUMMARY
Create Branch → Add → Commit → Pull → Push → PR → Merge

---

## Real-World Example

1. Create a new feature:
   git checkout -b feature/login

2. Make changes and stage:
   git add .

3. Commit changes:
   git commit -m "Add login feature"

4. Push to remote:
   git push origin feature/login

5. Create Pull Request and merge

---

## When to Use Commands

- `git fetch` → When you only want to download changes  
- `git pull` → When you want to download + merge  
- `git push` → When you want to upload your changes
