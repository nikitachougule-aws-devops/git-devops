## Git Components

### **Git Repository**

A Git repository contains files, history, and configuration managed by Git.

A Git repository is simply your project folder where Git tracks all changes.

👉 When you run: `git init`  
Git turns your normal folder into a repository.

---

### **.git Folder**

Inside every Git repository, there is a hidden folder called `.git`.

👉 This folder:

- Stores all your project history  
- Keeps track of commits, branches, and configurations  
- Is hidden to prevent accidental deletion  

⚠️ If you delete this folder, your entire Git history will be lost.

---

### **Remote Repository**

A remote repository is an online storage for your code.

👉 Examples: GitHub, GitLab  

👉 Why use it?

- Backup your code  
- Share with team members  
- Collaborate from anywhere  

---

## ✅ Summary

- **Git** helps you track and manage code changes  
- **Repository** = your project folder  
- **.git folder** = brain of Git (stores history)  
- **Remote repository** = cloud storage for your code
