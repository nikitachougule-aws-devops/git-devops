### 🔁 Git Reset vs Revert vs Rebase

### 🔹 Reset
- Moves HEAD to a previous commit  
- Deletes commits after that point (history changed)  
- Used to undo commits locally  

👉 Go back in time and erase history  

---

### 🔹 Revert
- Creates a new commit that undoes changes  
- Does NOT delete history  
- Safe for shared branches  

👉 Undo changes without deleting history  

---

### 🔹 Rebase
- Re-applies commits on top of another branch  
- Makes history clean and linear  
- Used before merging  

👉 Move your work on top of latest code  

---

## In short

- **Reset** → delete history  
- **Revert** → keep history, undo changes  
- **Rebase** → rewrite history cleanly  
---


## 🔥 Real-World Scenarios

### Scenario 1

You edited a file incorrectly  
👉 Use `restore`

---

### Scenario 2

You made a bad commit locally  
👉 Use `reset` (soft/mixed)

---

### Scenario 3

You pushed a bug to production  
👉 Use `revert` (safe option)

---

## Rules

- If the commit is already pushed → use **revert**  
- If it is local → you can use **reset**
